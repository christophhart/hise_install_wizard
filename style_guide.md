# Style guide for the web app

This file contains styling rules for the design of the web app. Please use this whenever making changes to the design or adding new elements.

## General page layout

- the content page should have a phase progress bar at the top followed by the content of the phase. At the bottom you will have three buttons for success, failure or skip this step
- there is a top bar that has a "HISE install wizard" label at the top and uses the image `images/logo_new.png` at the top left.

## Colours

- the background colour of the page is #222
- the background colour of elements is #333
- elements have rounded corners with 3px and a subtle border of 1px with #444
- UI elements that need an accent should use the HISE signal colour #90FFB1 
- every colour that indicates some kind of error should be #BB3434 
- every colour that indicates some kind of success should be #4E8E35
- every colour that indicates some kind of warning should be #FFBA00   
- code blocks should have a background colour of #111 and a text color of #999 
- add icon buttons for common button functionality like copy / paste or success & failure

## Font

- the normal font used should be "Lato"
- all code / monospace font should use "Source Sans Pro"
